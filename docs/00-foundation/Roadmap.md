# Roadmap

## Version 0.1.0 — Genesis

Status: active foundation design.

Genesis establishes the identity, architecture, and repository structure of PadMe AI Workstation as the reference implementation of the PadMe Engineering Platform.

### Goals

- Define the platform vision.
- Establish guiding principles.
- Document the initial architecture.
- Create the decision record process.
- Prepare the repository for long-term growth.

### Deliverables

- README
- Foundation documentation
- Architecture decision records
- Repository structure
- Initial roadmap

---

## Phase 1 — Developer Workstation

Goal: create the first reproducible local engineering environment.

Planned components:

- Visual Studio Code
- Git
- Docker
- OpenCode
- Gentle-AI
- Engram
- Ollama
- PowerShell automation baseline

---

## Phase 2 — Model Orchestration

Goal: define how PadMe selects and routes AI models.

Planned topics:

- GPT
- Claude
- Gemini
- Qwen
- DeepSeek
- Local models
- OpenRouter
- Model profiles
- Cost and quality tradeoffs

---

## Phase 3 — MCP Ecosystem

Goal: connect PadMe to tools and external systems through MCP-compatible interfaces.

Planned areas:

- GitHub
- Docker
- Databases
- Browser automation
- Microsoft 365
- SSH and servers
- Local AI

---

## Phase 4 — Agent Platform

Goal: create specialized agents with clear roles, boundaries, and capabilities.

Initial agents:

- PadMe Architect
- PadMe Senior Developer
- PadMe Code Reviewer
- PadMe Security
- PadMe Hotel IT
- PadMe Automation
- PadMe Documentation

---

## Phase 5 — Automation

Goal: convert repeated manual work into repeatable workflows.

Planned outputs:

- scripts
- playbooks
- templates
- reusable prompts
- operational procedures
- local automation helpers
